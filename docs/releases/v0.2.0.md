# v0.2.0 Release Notes

Status: release candidate / draft.

License: Apache-2.0.

## Highlights

- Driver Router for explicit browser, Tauri, Electron, and X11 path selection.
- Browser semantic driver powered by Playwright inside isolated Xvfb sessions.
- Experimental Electron semantic driver using Playwright Electron.
- Experimental Tauri WebDriver driver using `tauri-driver` when prerequisites and app configuration are available.
- Visual QA assertions for PNG screenshot diffing, region comparison, and before/after evidence.
- Visual baselines for local regression checks.
- Annotation-region visual assertions and pixel change containment checks.
- Optional noVNC live observer for watching isolated Xvfb sessions through localhost.
- Stronger evidence reports with screenshots, actions, annotations, visual diffs, baselines, and cleanup summaries.

## Optional or Dependency-Gated Checks

```text
pnpm smoke:observer
pnpm smoke:tauri-driver
```

`smoke:observer` passes when `x11vnc`, noVNC, and websockify dependencies are installed. It skips clearly when those optional packages are missing.

`smoke:tauri-driver` passes when Tauri WebDriver prerequisites and a compatible app configuration are available. It skips clearly when no Tauri app environment is configured.

## Local RC Verification Notes

- `smoke:observer` passed on the release-candidate Ubuntu machine after installing `x11vnc`, `novnc`, and `websockify`.
- `smoke:tauri-driver` reported an honest skip on the release-candidate machine because no Tauri app environment was configured. Tauri WebDriver prerequisites were available.

## Known Limitations

- Linux/X11/Xvfb-first.
- X11 fallback is coordinate-based.
- OCR is not implemented.
- Wayland backend is not implemented.
- Tauri and Electron semantic support is experimental.
- Live observer requires `x11vnc`, noVNC, and websockify.
- Visual QA is PNG-based and dimension-sensitive.
- Visual QA thresholds may need tuning for animations, antialiasing, browser chrome, and timestamps.
- Sessions are in-memory per server process.
- HTTP is local-only and unauthenticated by default.
- Live observer is local-only by default and should use SSH tunneling for remote viewing.
- Visual Annotation Handoff supports rectangle annotations in the MVP.
- Annotation repair smoke proves handoff and verification workflow, not guaranteed autonomous repair.

## Upgrade Notes From v0.1

The v0.1 Xvfb evidence path remains intact. Existing desktop, screenshot, HTTP, MCP, and annotation workflows continue to work. v0.2 adds semantic drivers, driver routing, Visual QA, visual baselines, and the optional live observer without replacing X11 fallback.

## Release Blockers

- Final verification run on the release machine.
- Optional observer dependencies may be installed later to turn `smoke:observer` from skipped to passed.
- Final release commit and tag.
